## 1. Контекст решения

В оригинальном PRD v2.3 (§2.5) реализован 4-шаговый детерминированный матчинг:
Страна+Тип → Сумма+Валюта → Дата±3 дня → Нечёткий контрагент.

По решению руководителя проекта алгоритм **упрощается** до реализуемого в рамках текущего спринта:

> *«Пусть сравнение идёт по номеру инвойса. В выписке будет указан в назначении перевода номер инвойса. Если там номера нет — тогда просто по названию контрагента. Всё. Остальные проверки пока не будем делать.»*

Дополнительно фиксируется: проверка по **сумме и валюте** остаётся как страховочный предфильтр во избежание ложных совпадений.

**Текущее состояние (WF-05, факт):** 12 колонок A–L.


|---|---|---|---|---|

### 3.2 Лист «Банк» — новый столбец

**Текущее состояние (WF-06, факт):** 13 колонок A–M.

```
A=bank_id | B=Банк    | C=Период | D=Страна | E=Статус
F=Дата документа      | G=Контрагент         | H=Дата оплаты
I=Сумма   | J=Валюта  | K=Описание
L=Статус сверки       | M=UUID реестра
```

**После изменения** — вставляется колонка **L**, существующие L и M сдвигаются:

```
A=bank_id | B=Банк    | C=Период | D=Страна | E=Статус
F=Дата документа      | G=Контрагент         | H=Дата оплаты
I=Сумма   | J=Валюта  | K=Описание
L=Номер инвойса (банк)   ← НОВАЯ
M=Статус сверки          ← было L
N=UUID реестра           ← было M
```

> ⚠️ В Google Sheets: вставить колонку перед текущей L («Статус сверки»). Проверить, что существующие данные в L и M корректно сдвинулись.

## 6. Алгоритм матчинга WF-07 (заменяет §2.5 PRD v2.3)

### 6.1 Входные данные

```
• Лист «Банк» (A–N): строки за период, Статус сверки (M) = PENDING или UNMATCHED
• Лист «Реестр» (A–M): строки за период, Статус оплаты (H) = «Оплачено»
```

### 6.2 Алгоритм

```
┌─────────────────────────────────────────────────────────────────┐
│  ШАГ 0. Предфильтр (обязательный)                               │
│                                                                  │
│   Банк[D].Страна  == Реестр[I].Страна                           │
│   Банк[E].Статус  == Реестр[B].Статус                           │
│   abs(Реестр[F].Сумма) ≈ Банк[I].Сумма  (допуск ±0.02)         │
│   Реестр[G].Валюта == Банк[J].Валюта    (без учёта регистра)    │
│                                                                  │
│   → 0 кандидатов → UNMATCHED, стоп                               │
└─────────────────────────────────────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────────────────────┐
│  ШАГ 1. Матчинг по номеру инвойса (приоритетный)                │
│                                                                  │
│   Если Банк[L] != '' И Реестр[M] != '':                         │
│     normalize(Банк[L]) == normalize(Реестр[M])                   │
│     (нижний регистр, убрать пробелы)                             │
│                                                                  │
│   → совпадение → MATCHED; UUID реестра → Банк[N]; стоп          │
│   → нет совпадения ИЛИ одно из полей пусто → Шаг 2              │
└─────────────────────────────────────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────────────────────┐
│  ШАГ 2. Матчинг по контрагенту (запасной)                       │
│                                                                  │
│   Если Банк[G] != '' И Реестр[D] != '':                         │
│     normalize(Банк[G]) == normalize(Реестр[D])                   │
│     normalize(): нижний регистр, убрать юр.формы                │
│     (ООО/ЗАО/ОАО/ЧУП/ИП/УП / sp.z o.o./S.A./Ltd/GmbH),        │
│     убрать спецсимволы и лишние пробелы                          │
│                                                                  │
│   → совпадение → MATCHED; UUID реестра → Банк[N]; стоп          │
│   → нет совпадения ИЛИ оба пустые → UNMATCHED                   │
└─────────────────────────────────────────────────────────────────┘
```

### 6.3 Допуск суммы: `±0.02` единицы валюты

Покрывает ошибки округления обеих банковских систем. Не создаёт ложных совпадений при типичных суммах малого бизнеса.

### 6.4 Граничные случаи

| Ситуация | Поведение |
|---|---|
| Банк: нет invoice_number и нет counterparty | UNMATCHED |
| Реестр: нет invoice_number, но есть counterparty | Шаг 1 → мимо; Шаг 2 → по контрагенту |
| invoice_number есть в банке, нет в реестре | Шаг 1 → мимо; Шаг 2 → по контрагенту |
| Сумма расходится > 0.02 | Не попадает в кандидаты (Шаг 0) |
| Статус оплаты = «Не оплачено» в реестре | Строка не участвует в матчинге |
| >1 кандидата после любого шага | MATCHED по первому + `[ambiguous]` в Описание |

### 6.5 JS-код для ноды матчинга (WF-07)

```javascript
const AMOUNT_TOLERANCE = 0.02;

function normalizeInvoice(s) {
  if (!s || String(s).trim() === '') return null;
  return String(s).toLowerCase().replace(/\s+/g, '');
}

function normalizeCounterparty(s) {
  if (!s || String(s).trim() === '') return null;
  const forms = [
    'ооо','зао','оао','одо','чуп','уп','ип',
    'sp z o o','spzoo','s a','sp j','sp k',
    'ltd','gmbh','inc','llc','sarl','sas'
  ];
  let n = String(s).toLowerCase()
    .replace(/[^a-zа-яёa-z0-9\s]/gi, ' ')
    .replace(/\s+/g, ' ').trim();
  forms.forEach(f => { n = n.replace(new RegExp(`\\b${f}\\b`, 'g'), ''); });
  return n.replace(/\s+/g, ' ').trim() || null;
}

// bankRows — из листа «Банк» (колонки по заголовкам), за период, PENDING/UNMATCHED
// regRows  — из листа «Реестр», за период, Статус оплаты = «Оплачено»

const results = [];

for (const tx of bankRows) {
  const out = { ...tx, 'Статус сверки': 'UNMATCHED', 'UUID реестра': '' };

  // Шаг 0: предфильтр
  let cands = regRows.filter(r =>
    r['Страна'] === tx['Страна'] &&
    r['Статус'] === tx['Статус'] &&
    String(r['Валюта']).toUpperCase() === String(tx['Валюта']).toUpperCase() &&
    Math.abs(Math.abs(parseFloat(r['Сумма'])) - parseFloat(tx['Сумма'])) <= AMOUNT_TOLERANCE
  );
  if (!cands.length) { results.push(out); continue; }

  // Шаг 1: по инвойсу
  const bInv = normalizeInvoice(tx['Номер инвойса (банк)']);
  if (bInv) {
    const hit = cands.filter(r => normalizeInvoice(r['Номер инвойса']) === bInv);
    if (hit.length) {
      out['Статус сверки'] = 'MATCHED';
      out['UUID реестра']  = hit[0]['UUID'];
      if (hit.length > 1) out['Описание'] = (out['Описание'] || '') + ' [ambiguous]';
      results.push(out); continue;
    }
  }

  // Шаг 2: по контрагенту
  const bCP = normalizeCounterparty(tx['Контрагент']);
  if (bCP) {
    const hit = cands.filter(r => normalizeCounterparty(r['Контрагент']) === bCP);
    if (hit.length) {
      out['Статус сверки'] = 'MATCHED';
      out['UUID реестра']  = hit[0]['UUID'];
      if (hit.length > 1) out['Описание'] = (out['Описание'] || '') + ' [ambiguous]';
      results.push(out); continue;
    }
  }

  results.push(out);
}

return results.map(r => ({ json: r }));
```
